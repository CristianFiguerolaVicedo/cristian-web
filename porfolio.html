Restaurant finder

The main function of this app is finding nearby restaurants, sorting them by rating, distance and type of food. Also it is possible to write reviews.

This app, as most of the ones I make it's made using Springboot for the backend, PostgreSQL as the database and React for the frontend.

Money Manager

This web is made to keep track of expenses and incomes in an easy and simple way.

The application has many views. The main one is the dashboard, in which you can see a graphic to see the information in a more visual and appealing way as well as the latest incomes, expenses and all the transactions.

Another views are the ones showing the incomes and expenses, where there is a list and a linear graphic to show the tendency of the current month.

Finally, the last page is the one where you can filter the transactions by income or expense, start date and end date, the field, the sort order and a keyword.

This app, as most of the ones I make it's made using Springboot for the backend, PostgreSQL as the database and React for the frontend.

Inventory

This web surged as an alternative to conventional applications used for keeping inventory that little and medium e-commerces use such as excel and spreadsheets in general.

This web has some pages that we are going to go through now:

When opening the web, the first thing that you can see is obviously the login and register page, but after logging in we can first see the list of products with their respective information.

After the products, we have the categories page, which is quite simple, but helps us filter the products by their category to make it easier to find the product we are looking for.

Next we have the sales page, where we can see all the sales made at the current month with the information of the sale and the products sold.

Next we have the filter page, which, as I said before, helps us look for an specific product without having to go through every product by filtering by the date that the product was created, the sort field and order and a keyword.

The next thing we have is a simple calendar to see and create important events we have to keep in mind.

Finally, we have a balance page wherer we have the total balance of the month we want to select counting the incomes and expenses that we create and the sales of that month.

This app, as most of the ones I make it's made using Springboot for the backend, PostgreSQL as the database and React for the frontend.
